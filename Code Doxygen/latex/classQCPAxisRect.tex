\doxysubsection{Detailed Description}
Holds multiple axes and arranges them in a rectangular shape. 

This class represents an axis rect, a rectangular area that is bounded on all sides with an arbitrary number of axes.

Initially \doxylink{classQCustomPlot}{QCustom\+Plot} has one axis rect, accessible via \doxylink{classQCustomPlot_ae5eefcb5f6ca26689b1fd4f6e25b42f9}{QCustom\+Plot\+::axis\+Rect()}. However, the layout system allows to have multiple axis rects, e.\+g. arranged in a grid layout (\doxylink{classQCustomPlot_af1a1f1f571237deb7c2bd34a5e9f018f}{QCustom\+Plot\+::plot\+Layout}).

By default, \doxylink{classQCPAxisRect}{QCPAxis\+Rect} comes with four axes, at bottom, top, left and right. They can be accessed via \doxylink{classQCPAxisRect_a583ae4f6d78b601b732183f6cabecbe1}{axis} by providing the respective axis type (\doxylink{classQCPAxis_ae2bcc1728b382f10f064612b368bc18a}{QCPAxis\+::\+Axis\+Type}) and index. If you need all axes in the axis rect, use \doxylink{classQCPAxisRect_a8db4722cb93e9c4a6f0d91150c200867}{axes}. The top and right axes are set to be invisible initially (\doxylink{classQCPLayerable_a3bed99ddc396b48ce3ebfdc0418744f8}{QCPAxis\+::set\+Visible}). To add more axes to a side, use \doxylink{classQCPAxisRect_a2a845ee60eac7a8d5e71718c5e77c274}{add\+Axis} or \doxylink{classQCPAxisRect_a792e1f3d9cb1591fca135bb0de9b81fc}{add\+Axes}. To remove an axis, use \doxylink{classQCPAxisRect_a03c39cd9704f0d36fb6cf980cdddcbaa}{remove\+Axis}.

The axis rect layerable itself only draws a background pixmap or color, if specified (\doxylink{classQCPAxisRect_af615ab5e52b8e0a9a0eff415b6559db5}{set\+Background}). It is placed on the "{}background"{} layer initially (see \doxylink{classQCPLayer}{QCPLayer} for an explanation of the \doxylink{classQCustomPlot}{QCustom\+Plot} layer system). The axes that are held by the axis rect can be placed on other layers, independently of the axis rect.

Every axis rect has a child layout of type \doxylink{classQCPLayoutInset}{QCPLayout\+Inset}. It is accessible via \doxylink{classQCPAxisRect_a949f803466619924c7018df4b511ae10}{inset\+Layout} and can be used to have other layout elements (or even other layouts with multiple elements) hovering inside the axis rect.

If an axis rect is clicked and dragged, it processes this by moving certain axis ranges. The behaviour can be controlled with \doxylink{classQCPAxisRect_ae6aef2f7211ba6097c925dcd26008418}{set\+Range\+Drag} and \doxylink{classQCPAxisRect_a648cce336bd99daac4a5ca3e5743775d}{set\+Range\+Drag\+Axes}. If the mouse wheel is scrolled while the cursor is on the axis rect, certain axes are scaled. This is controllable via \doxylink{classQCPAxisRect_a7960a9d222f1c31d558b064b60f86a31}{set\+Range\+Zoom}, \doxylink{classQCPAxisRect_a9442cca2aa358405f39a64d51eca13d2}{set\+Range\+Zoom\+Axes} and \doxylink{classQCPAxisRect_a895d7ac745ea614e04056244b3c138ac}{set\+Range\+Zoom\+Factor}. These interactions are only enabled if \doxylink{classQCustomPlot_a5ee1e2f6ae27419deca53e75907c27e5}{QCustom\+Plot\+::set\+Interactions} contains \doxylink{namespaceQCP_a2ad6bb6281c7c2d593d4277b44c2b037a2c4432b9aceafb94000be8d1b589ef18}{QCP\+::i\+Range\+Drag} and \doxylink{namespaceQCP_a2ad6bb6281c7c2d593d4277b44c2b037abee1e94353525a636aeaf0ba32b72e14}{QCP\+::i\+Range\+Zoom}.

Adds a new axis to the axis rect side specified with {\itshape type}, and returns it. If {\itshape axis} is 0, a new \doxylink{classQCPAxis}{QCPAxis} instance is created internally. \doxylink{classQCustomPlot}{QCustom\+Plot} owns the returned axis, so if you want to remove an axis, use \doxylink{classQCPAxisRect_a03c39cd9704f0d36fb6cf980cdddcbaa}{remove\+Axis} instead of deleting it manually.

You may inject \doxylink{classQCPAxis}{QCPAxis} instances (or subclasses of \doxylink{classQCPAxis}{QCPAxis}) by setting {\itshape axis} to an axis that was previously created outside \doxylink{classQCustomPlot}{QCustom\+Plot}. It is important to note that \doxylink{classQCustomPlot}{QCustom\+Plot} takes ownership of the axis, so you may not delete it afterwards. Further, the {\itshape axis} must have been created with this axis rect as parent and with the same axis type as specified in {\itshape type}. If this is not the case, a debug output is generated, the axis is not added, and the method returns {\ttfamily nullptr}.

This method can not be used to move {\itshape axis} between axis rects. The same {\itshape axis} instance must not be added multiple times to the same or different axis rects.

If an axis rect side already contains one or more axes, the lower and upper endings of the new axis (\doxylink{classQCPAxis_a08af1c72db9ae4dc8cb8a973d44405ab}{QCPAxis\+::set\+Lower\+Ending}, \doxylink{classQCPAxis_a69119b892fc306f651763596685aa377}{QCPAxis\+::set\+Upper\+Ending}) are set to \doxylink{classQCPLineEnding_a5ef16e6876b4b74959c7261d8d4c2cd5a126c390f0c359fcd8df1fc5e38d26d5b}{QCPLine\+Ending\+::es\+Half\+Bar}.

This event gets called when the user turns the mouse scroll wheel while the cursor is over the layerable. Whether a cursor is over the layerable is decided by a preceding call to \doxylink{classQCPLayoutElement_a62d60f1b0ee9bb23e38811d71586d79c}{select\+Test}.

The current pixel position of the cursor on the \doxylink{classQCustomPlot}{QCustom\+Plot} widget is accessible via {\ttfamily event-\/\texorpdfstring{$>$}{>}pos()}.

The {\ttfamily event-\/\texorpdfstring{$>$}{>}angle\+Delta()} indicates how far the mouse wheel was turned, which is usually +/-\/ 120 for single rotation steps. However, if the mouse wheel is turned rapidly, multiple steps may accumulate to one event, making the delta larger. On the other hand, if the wheel has very smooth steps or none at all, the delta may be smaller.

The default implementation does nothing.
