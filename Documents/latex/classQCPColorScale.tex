\doxysubsection{Detailed Description}
A color scale for use with color coding data such as \doxylink{classQCPColorMap}{QCPColor\+Map}. 

This layout element can be placed on the plot to correlate a color gradient with data values. It is usually used in combination with one or multiple \doxylink{classQCPColorMap}{QCPColor\+Maps}.



The color scale can be either horizontal or vertical, as shown in the image above. The orientation and the side where the numbers appear is controlled with \doxylink{classQCPColorScale_a1bf9bdb291927c422dd66b404b206f1f}{set\+Type}.

Use \doxylink{classQCPColorMap_aa828921db364fe3c6af4619580ab85fd}{QCPColor\+Map\+::set\+Color\+Scale} to connect a color map with a color scale. Once they are connected, they share their gradient, data range and data scale type (\doxylink{classQCPColorScale_a1f29583bb6f1e7f473b62fb712be3940}{set\+Gradient}, \doxylink{classQCPColorScale_abe88633003a26d1e756aa74984587fef}{set\+Data\+Range}, \doxylink{classQCPColorScale_aeb6107d67dd7325145b2498abae67fc3}{set\+Data\+Scale\+Type}). Multiple color maps may be associated with a single color scale, to make them all synchronize these properties.

{\footnotesize\ttfamily void QCPColor\+Scale\+::wheel\+Event (\begin{DoxyParamCaption}\item[{QWheel\+Event \texorpdfstring{$\ast$}{*}}]{event}{}\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [protected]}, {\ttfamily [virtual]}}

This event gets called when the user turns the mouse scroll wheel while the cursor is over the layerable. Whether a cursor is over the layerable is decided by a preceding call to \doxylink{classQCPLayoutElement_a62d60f1b0ee9bb23e38811d71586d79c}{select\+Test}.

The current pixel position of the cursor on the \doxylink{classQCustomPlot}{QCustom\+Plot} widget is accessible via {\ttfamily event-\/\texorpdfstring{$>$}{>}pos()}.

The {\ttfamily event-\/\texorpdfstring{$>$}{>}angle\+Delta()} indicates how far the mouse wheel was turned, which is usually +/-\/ 120 for single rotation steps. However, if the mouse wheel is turned rapidly, multiple steps may accumulate to one event, making the delta larger. On the other hand, if the wheel has very smooth steps or none at all, the delta may be smaller.

The default implementation does nothing.
